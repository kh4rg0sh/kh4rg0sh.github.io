## Explanation

**Lemma 1.** If it is possible to transform both arrays into the same array, then it is possible to transform both arrays into their sorted order.

**Proof.**

The allowed operation only swaps elements within an array, so it preserves the multiset of values.

Therefore, if both arrays can be transformed into some common array, they must contain exactly the same multiset of elements. Consequently, both arrays can be transformed into the unique sorted arrangement of that multiset.

Hence it is sufficient to determine whether both arrays can be transformed into their sorted order.

---

**Lemma 2.** Every swap changes the parity of the inversion count.

**Proof.**

A swap can be decomposed into an odd number of adjacent swaps. Since each adjacent swap changes the inversion count by exactly one, every swap flips the parity of the inversion count.

Therefore, the parity of the number of swaps required to reach the sorted array is exactly the parity of the inversion count.

---

**Lemma 3.** If the arrays contain the same multiset of values and their inversion parities are equal, then both arrays can be transformed into their sorted order.

**Proof.**

Let

$$
p_A
$$

denote the parity of the inversion count of array $A$, and similarly define

$$
p_B.
$$

The parity of the number of swaps needed to sort an array is equal to its inversion parity.

If

$$
p_A = p_B,
$$

then both arrays require swap sequences of the same parity to reach the sorted array. Therefore, both arrays can be transformed into the same sorted arrangement.

Conversely, if the parities differ, then one array requires an even number of swaps while the other requires an odd number, making it impossible for both transformation sequences to be compatible.

Thus equality of inversion parity is both necessary and sufficient.

### Algorithm

1. Sort both arrays.
2. If the sorted arrays are different, output `NO`.
3. Otherwise compute the inversion parity of both arrays.
4. If the parities are equal, output `YES`; otherwise output `NO`.

The inversion count can be computed in $\mathcal{O}(n \log n)$ using an order-statistics tree.

## Code

```cpp
void solve() {
    i64 n;
    cin >> n;

    vector<i64> a(n + 1), b(n + 1);

    for (int i = 1; i <= n; i++) cin >> a[i];
    for (int i = 1; i <= n; i++) cin >> b[i];

    vector<i64> A = a, B = b;

    sort(A.begin() + 1, A.end());
    sort(B.begin() + 1, B.end());

    if (A != B) {
        cout << "NO\n";
    } else {
        i64 p1 = 0, p2 = 0;

        ordered_set_64 tree;

        for (int i = 1; i <= n; i++) {
            p1 += tree.size() - tree.order_of_key(a[i] + 1);
            tree.insert(a[i]);
        }

        tree.clear();

        for (int i = 1; i <= n; i++) {
            p2 += tree.size() - tree.order_of_key(b[i] + 1);
            tree.insert(b[i]);
        }

        if ((p1 % 2) == (p2 % 2)) {
            cout << "YES\n";
        } else {
            cout << "NO\n";
        }
    }
}
