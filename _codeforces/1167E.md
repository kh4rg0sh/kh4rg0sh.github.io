## Explanation

**Definition.** A pair $(l,r)$ is called **valid** if deleting all values satisfying

$$
l \le a_i \le r
$$

from the array leaves a non-decreasing array.

**Definition.** A number $p$ is called **prefix-good** if the pair

$$
(p+1,x)
$$

is valid.

**Lemma 1.** If a pair $(l,r)$ is valid, then both $(l-1,r)$ and $(l,r+1)$ are valid.

**Proof.**

If $(l,r)$ is valid, then the remaining sequence has the form

$$
1 \cdots 1
\mid
2 \cdots 2
\mid
\cdots
\mid
(l-1)
\mid
(r+1)
\cdots
\mid
x \cdots x.
$$

Removing either $(l-1)$ or $(r+1)$ from an already non-decreasing sequence preserves the non-decreasing property. Therefore $(l-1,r)$ and $(l,r+1)$ are also valid.

**Lemma 2.** If $p$ is prefix-good, then every $q < p$ is also prefix-good. Similarly, if $p$ is not prefix-good, then no $q > p$ can be prefix-good.

**Proof.**

If $p$ is prefix-good, then the values $1,\ldots,p$ appear in non-decreasing order in the array. Any prefix of a non-decreasing sequence is also non-decreasing, so every $q<p$ is prefix-good.

Conversely, if $p$ is not prefix-good, then the values $1,\ldots,p$ do not appear in non-decreasing order. Extending this set to $1,\ldots,q$ for any $q>p$ cannot restore the property. Hence no larger value can be prefix-good.

**Lemma 3.** If $p$ is not prefix-good, then for any

$$
p < l \le r,
$$

the pair $(l,r)$ cannot be valid.

**Proof.**

If $(l,r)$ were valid, then $(l-1)$ would have to be prefix-good. Since $l-1 \ge p$ and $p$ is not prefix-good, this is impossible.

### Back to the Problem

1. Compute the first and last occurrence of every value $1 \le i \le x$. This takes $\mathcal{O}(n)$ time.

2. Compute the maximum prefix-good value $p$. We only need to verify that consecutive values appear without overlap:

$$
\operatorname{last}[i-1]
<
\operatorname{first}[i].
$$

3. Similarly, compute the minimum suffix-good value $s$ (equivalently, prefix-good on the reversed value ordering). This is determined by

$$
\operatorname{last}[i]
<
\operatorname{first}[i+1].
$$

4. If $p \ge s$, then necessarily $p=x$ and $s=1$. In this case every pair $(l,r)$ is valid. Otherwise we have $p<s$ and continue.

5. Any valid pair must satisfy

$$
1 \le l \le p+1
$$

and

$$
s-1 \le r \le x.
$$

Otherwise the remaining sequence cannot be non-decreasing.

6. For each $l$, define $g(l)$ as the minimum valid value of $r$. Then the answer is

$$
\sum_{i=1}^{p+1}
\left(
x-(g(i)-1)
\right).
$$

7. To compute $g(i)$, let

$$
m =
\max(a_1,\ldots,a_{\operatorname{last}[i-1]}).
$$

Then $r$ must be at least $m$, and also satisfy the previous constraints. Therefore

$$
r \ge \max(i,s-1,m),
$$

which gives

$$
g(i)
=
\max\!\Bigl(
i,
s-1,
\max([1,\operatorname{last}[i-1]])
\Bigr).
$$

This allows all values of $g(i)$ to be computed efficiently.

## Code

```cpp
void solve() {
    ll n, x;
    std::cin >> n >> x;

    ll a[n + 1];
    for (ll i = 1; i <= n; i++) {
        std::cin >> a[i];
    }

    std::vector<ll> first(x + 2, 0LL), last(x + 2, 0LL);

    for (ll i = 1; i <= n; i++) {
        if (first[a[i]] == 0) {
            first[a[i]] = i;
            last[a[i]] = i;
        } else {
            last[a[i]] = i;
        }
    }

    ll b[n + 1];
    for (ll i = 1; i <= n; i++) {
        b[i] = a[i];
    }

    std::sort(b + 1, b + n + 1);

    ll p = b[1], s = b[n];

    for (ll i = 2; i <= n; i++) {
        if (b[i - 1] == b[i]) {
            if (i == n) {
                p = x;
            }
            continue;
        }

        if (last[b[i - 1]] < first[b[i]]) {
            p = b[i];

            if (i == n) {
                p = x;
            }
        } else {
            p = b[i] - 1;
            break;
        }
    }

    for (ll i = n - 1; i > 0; i--) {
        if (b[i] == b[i + 1]) {
            if (i == 1) {
                s = 1;
            }
            continue;
        }

        if (last[b[i]] < first[b[i + 1]]) {
            s = b[i];

            if (i == 1) {
                s = 1;
            }
        } else {
            s = b[i] + 1;
            break;
        }
    }

    ll ans = 0;

    ll mx[n + 1];
    mx[0] = 0;

    for (ll i = 1; i <= n; i++) {
        mx[i] = std::max(mx[i - 1], a[i]);
    }

    for (ll i = 1; i <= x; i++) {
        if (last[i] == 0) {
            last[i] = last[i - 1];
        }
    }

    for (ll i = 1; i <= std::min(x, p + 1); i++) {
        ans += (x + 1 - std::max({i, s - 1, mx[last[i - 1]]}));
    }

    std::cout << ans << '\n';
}
```
