## Explanation
Let $d(p)$ denote the distance from position $p$ to its nearest shop.

We first sort all shop positions. Consider any interval between two consecutive shops $[L, R]$. For every position inside this interval,

$$
d(p) = \min(p - L, R - p).
$$

Therefore, the positions with maximum value of $d(p)$ are precisely the middle positions of the interval. If the interval length is even, there is a unique middle position. Otherwise, there are two middle positions and both attain the same optimal value.

For every interval, we insert its middle position(s) into a priority queue. For a position $p$, we also store the endpoint of the interval that is nearest to it. Let this endpoint be denoted by $near$.

Observe that once a middle position is chosen, the remaining positions of that interval appear in decreasing order of distance as we move away from the center. More precisely, if $p$ is generated from endpoint $near$, then the next best ungenerated position on the same side is:

* $p - 1$ if $p > near$,
* $p + 1$ if $p < near$.

Its distance decreases by exactly one:

$$
d(p \pm 1) = d(p) - 1.
$$

Hence every interval generates a sequence of positions sorted by decreasing distance to the nearest shop:

$$
m,; m-1,; m+1,; m-2,; m+2,;\ldots
$$

where $m$ denotes a middle position of the interval. One of the directions may be absent near the boundaries.

The priority queue always stores the largest currently available distance from every interval. Whenever we extract a position, we immediately insert the next position from the same sequence. Thus, each interval behaves like a sorted list of candidate positions ordered by decreasing value of $d(p)$.

The priority queue therefore performs a $k$-way merge of these sorted lists. Consequently, positions are extracted globally in non-increasing order of distance to the nearest shop.

Therefore, after selecting the first $k$ distinct extracted positions, we obtain exactly the $k$ positions having the largest possible values of $d(p)$. This maximizes the distance from every chosen pole to its nearest shop, proving the correctness of the greedy algorithm.

```c++
void solve() {
    i64 n, k, x; cin >> n >> k >> x;
    vector<i64> a(n + 1);
    for (int i = 1; i <= n; i++) cin >> a[i];
    sort(a.begin() + 1, a.end());

    using T = array<i64, 3>;
    priority_queue<T> pq;

    for (int i = 1; i <= n; i++) {
        if (i == 1) {
            pq.push({a[i], 0, a[i]});
        } else {
            i64 len = a[i] - a[i - 1];
            if (len % 2) {
                i64 mid = a[i - 1] + len / 2;
                pq.push({abs(mid - a[i - 1]), mid, a[i - 1]});
                pq.push({abs(mid + 1 - a[i]), mid + 1, a[i]});
            } else {
                i64 mid = a[i - 1] + len / 2;
                pq.push({abs(mid - a[i - 1]), mid, a[i - 1]});
                pq.push({abs(mid - a[i]), mid, a[i]});
            }
        }
    }
    pq.push({abs(x - a[n]), x, a[n]});

    i64 ans = 1e18;
    map<i64, bool> vis;
    while (pq.size() && k) {
        auto [val, tel, near] = pq.top(); pq.pop();
        if (!vis.count(tel)) k--;
        ans = min(ans, val);
        vis[tel] = true;

        if (tel > near) {
            if (tel - 1 - near >= 0)
                pq.push({abs(tel - 1 - near), tel - 1, near});
        }
        if (tel < near) {
            if (near - tel - 1 >= 0)
                pq.push({abs(near - tel - 1), tel + 1, near});
        }
    }
    if (k > 0) {
        for (int i = 0; i <= x; i++) {
            cout << i << ' ';
        }
        cout << '\n';
        return;
    }
    for (auto& [u, v]: vis) {
        cout << u << ' ';
    }
    cout << '\n';
}
```
