## Explanation

**Lemma:** There are only $30$ powers of two less than $2 \times 10^9$.

Let this set be denoted by $\mathbb{P}$.

We want to compute

$$
\sum_{i=1}^{n}\sum_{j=i+1}^{n}[a_i+a_j\in\mathbb{P}]
$$

This can be rewritten as

$$
\frac{1}{2}
\left(
\sum_{1\le i,j\le n}
[a_i+a_j\in\mathbb{P}]
-
\sum_{1\le i\le n}
[2a_i\in\mathbb{P}]
\right)
$$

Iterating over all pairs $(i,j)$ would require $\mathcal{O}(n^2)$ time.

Instead, for every element $a_i$, we iterate over all powers of two in $\mathbb{P}$ and check whether the complement

$$
\text{power} - a_i
$$

exists in the array.

Since there are only $30$ relevant powers of two, the complexity becomes

$$
\mathcal{O}(30n\log n)
$$

when using an ordered map.

## Code

```cpp
void solve() {
    ll n;
    std::cin >> n;

    std::vector<ll> a(n + 1);
    std::map<ll, ll> mp;

    for (ll i = 1; i <= n; i++) {
        std::cin >> a[i];
        mp[a[i]]++;
    }

    std::vector<ll> powers;
    for (ll i = 2; i <= 2 * 1e9; i *= 2) {
        powers.push_back(i);
    }

    ll ans = 0;

    for (ll i = 1; i <= n; i++) {
        for (auto sum : powers) {
            if (mp.count(sum - a[i])) {
                ans += mp[sum - a[i]];

                if (sum == 2 * a[i]) {
                    ans--;
                }
            }
        }
    }

    ans /= 2;
    std::cout << ans << '\n';
}
```

## Performance Optimizations

Suppose we replace

```cpp
if (mp.count(sum - a[i])) {
    ans += mp[sum - a[i]];

    if (sum == 2 * a[i]) {
        ans--;
    }
}
```

with

```cpp
ans += mp[sum - a[i]];

if (sum == 2 * a[i]) {
    ans--;
}
```

This results in roughly a **10× slower runtime**.

The reason is how `std::map::operator[]` works. According to the GCC documentation:

> If the key does not exist, a pair with that key is created using default values, which is then returned.

As a result, every lookup for a missing key inserts a new entry into the map, causing significant overhead.
