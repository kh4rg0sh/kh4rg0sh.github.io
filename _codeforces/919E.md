## Explanation

**Lemma.** For $k \not\equiv 0 \pmod p$, the map

$$
x \mapsto kx \pmod p
$$

is a bijection.

Suppose we iterate over the possible values of $n \bmod (p-1)$ and let

$$
n \equiv i \pmod{p-1}.
$$

Then from

$$
na^n \equiv b \pmod p
$$

we obtain

$$
n \equiv ba^{-i} \equiv j \pmod p.
$$

Thus, for each residue $i$, we obtain the system

$$
n \equiv j \pmod p,
$$

$$
n \equiv i \pmod{p-1},
$$

where

$$
j \equiv ba^{-i} \pmod p.
$$

Since $p$ and $p-1$ are coprime, we can apply the Chinese Remainder Theorem:

$$
n \equiv
ip\Bigl(p^{-1}\bmod (p-1)\Bigr)
+
j(p-1)\Bigl((p-1)^{-1}\bmod p\Bigr)
\pmod{p(p-1)}.
$$

Using

$$
p^{-1}\equiv 1 \pmod{p-1}
\quad\text{and}\quad
(p-1)^{-1}\equiv -1 \pmod p,
$$

this simplifies to

$$
n
\equiv
ip-j(p-1)
\pmod{p(p-1)}.
$$

Let

$$
y \equiv ip-j(p-1) \pmod{p(p-1)}.
$$

We now need to count how many integers $n$ in the range $[1,x]$ satisfy

$$
n \equiv y \pmod{p(p-1)}.
$$

The count is

$$
\left\lfloor
\frac{x}{p(p-1)}
\right\rfloor
+
[y \le x \bmod p(p-1)].
$$

Repeating this process for all $i \in [0,p-2]$ yields an $\mathcal{O}(p)$ algorithm.

## Code

```cpp
ll binpow(ll x, ll y, ll m) {
    x %= m;
    ll result = 1;

    while (y > 0) {
        if (y & 1) result = result * x % m;
        x = x * x % m;
        y >>= 1;
    }

    result %= m;
    return result;
}

ll modinv(ll x, ll p) {
    return binpow(x, p - 2, p);
}

void solve() {
    ll a, b, p, x;
    std::cin >> a >> b >> p >> x;

    ll ans = 0;
    ll val = 1;

    for (ll i = 0; i < p - 1; i++, val = (val * a) % p) {
        ll j = (b * modinv(val, p)) % p;

        ll n = -j * (p - 1) + i * p;
        if (n < 0) {
            n += p * (p - 1);
        }

        ans += x / (p * (p - 1));

        if (n <= x % (p * (p - 1))) {
            ans++;
        }
    }

    std::cout << ans << '\n';
}
```
