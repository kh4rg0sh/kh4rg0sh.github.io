## Explanation

Construct a complete binary tree whose leaves correspond to the cows. Every internal node represents a battle between its two children.

For a node representing an interval $[l,r]$, let

$$
m = \frac{l+r}{2}.
$$

The left child represents $[l,m]$ and the right child represents $[m+1,r]$.

Define

$$
\text{winner}(v)=
\begin{cases}
0 & \text{if the left child wins},\\
1 & \text{if the right child wins}.
\end{cases}
$$

This naturally induces a binary string on the path from the root to any leaf.

---

### Position Encoding

**Lemma 1.** Let a node be at depth $i$ (root has depth $0$). If the winner of that node is the child containing a given cow, then the cow moves above all cows contained in the opposite subtree.

The number of cows in the opposite subtree equals

$$
2^{n-i-1}.
$$

Therefore choosing the winning child at depth $i$ contributes an offset of

$$
2^{n-i-1}
$$

to the final position of the cow.

**Proof.**

Every internal node corresponds to a tournament bracket. If the cow belongs to the winning half of a node, then it is ranked above every cow in the losing half.

Since the subtree size at depth $i$ equals

$$
2^{n-i},
$$

each child contains

$$
2^{n-i-1}
$$

cows.

Hence the cow is placed above exactly

$$
2^{n-i-1}
$$

additional cows whenever its side wins at that level.
$\square$

---

**Lemma 2.** Let $b_i \in \{0,1\}$ denote whether the cow belongs to the winning child at depth $i$. Then the number of cows ranked below it is

$$
\sum_i b_i 2^{n-i-1}.
$$

Consequently its $1$-indexed position equals

$$
1 + \sum_i b_i 2^{n-i-1}.
$$

**Proof.**

By Lemma 1, each level contributes independently to the number of cows below the current cow.

Summing all contributions over the path from the leaf to the root gives the total number of cows ranked below it.

Adding one converts this quantity into a $1$-indexed position.
$\square$

Therefore the position of any cow can be computed by walking from its leaf to the root and accumulating the corresponding offsets.

Since the height of the tree is $n$, this requires $\mathcal{O}(n)$ time.

---

### Processing Queries

Suppose cow $b$ changes value from

$$
a_b
$$

to

$$
c.
$$

Observe that only battles on the path from leaf $b$ to the root can change.

Every other node depends only on segments whose XOR values remain unchanged.

---

**Lemma 3.** After modifying cow $b$, only ancestors of leaf $b$ can change their winner.

**Proof.**

The value of a battle depends only on the XOR of the two halves represented by that node.

Changing $a_b$ affects exactly those segments containing index $b$.

These segments are precisely the ancestors of leaf $b$.

Hence all other nodes keep the same winner.
$\square$

For every ancestor we recompute the winner after applying the XOR difference

$$
a_b \oplus c.
$$

If the new winner coincides with the side containing cow $b$, then the cow gains the offset associated with that node.

Otherwise it gains nothing.

Summing these contributions over all ancestors yields the number of cows that will be ranked above the modified cow.

Since the root-to-leaf path contains exactly $n$ nodes, each query requires only $\mathcal{O}(n)$ time.

---

### Complexity

Building the tournament tree takes

$$
\mathcal{O}(2^n).
$$

Each query traverses a single root-to-leaf path and performs constant work per level, giving

$$
\mathcal{O}(n)
$$

time per query.

Therefore the total complexity is

$$
\mathcal{O}(2^n + qn).
$$

## Code

```cpp
void solve() {
    i64 n, q; cin >> n >> q;

    vector<i64> a((1LL << n) + 1);
    for (int i = 1; i <= (1LL << n); i++) {
        cin >> a[i];
    }

    vector<i64> pref((1LL << n) + 1);
    for (int i = 1; i <= (1LL << n); i++) {
        pref[i] = pref[i - 1] ^ a[i];
    }

    struct BinaryNode {
        i64 l, r;
        i64 better, offset;
        BinaryNode *left, *right, *parent;
    };

    BinaryNode root = {
        .l = 1,
        .r = (1LL << n),
        .offset = (1LL << n) / 2
    };

    auto battle = [&] (i64 l, i64 mid, i64 r) -> i64 {
        i64 first = pref[mid] ^ pref[l - 1];
        i64 second = pref[r] ^ pref[mid];

        if (first >= second) return 0;
        return 1;
    };

    map<i64, BinaryNode*> mp;

    auto dfs = [&] (auto&& self, BinaryNode& cur) -> void {
        i64 l = cur.l;
        i64 r = cur.r;

        if (l == r) {
            mp[l] = &cur;
            return;
        }

        i64 len = (r - l + 1) / 2;
        i64 mid = l + len - 1;

        BinaryNode* left = new BinaryNode;
        BinaryNode* right = new BinaryNode;

        left->l = l;
        left->r = mid;

        right->l = mid + 1;
        right->r = r;

        cur.better = battle(l, mid, r);

        cur.left = left;
        cur.right = right;

        left->parent = &cur;
        right->parent = &cur;

        left->offset = cur.offset / 2;
        right->offset = left->offset;

        self(self, *left);
        self(self, *right);
    };

    dfs(dfs, root);

    while (q--) {
        i64 b, c;
        cin >> b >> c;

        i64 offset = 0;
        BinaryNode* cur = mp[b];

        while (cur->parent != nullptr) {
            BinaryNode* parent = cur->parent;

            i64 side = (parent->right == cur);

            i64 l = parent->l;
            i64 r = parent->r;

            i64 len = (r - l + 1) / 2;
            i64 mid = l + len - 1;

            i64 first = pref[mid] ^ pref[l - 1];
            i64 second = pref[r] ^ pref[mid];

            if (side) {
                second ^= (a[b] ^ c);
            } else {
                first ^= (a[b] ^ c);
            }

            i64 new_val = (first < second);

            if (new_val != side) {
                offset += parent->offset;
            }

            cur = parent;
        }

        cout << offset << '\n';
    }
}
